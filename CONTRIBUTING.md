# Contributing to Agentic Workflow Demos

Thank you for your interest in contributing! Please follow these guidelines to ensure a smooth review process.

## Pull Request Requirements

1. **Descriptive Title**: PR titles must clearly describe the change. Avoid vague titles like "fix", "update", or "changes".
2. **Detailed Description**: Every PR must include a description explaining:
   - **What** was changed
   - **Why** the change is needed
   - **How** it was tested
3. **Focused Changes**: Each PR should address a single concern. Do not mix unrelated changes in one PR.
4. **Small PRs**: Keep PRs under 500 lines of changes when possible. Large PRs are harder to review.
5. **Tests Required**: New features and bug fixes must include tests.
6. **No Secrets**: Never commit API keys, passwords, or credentials.

## Code Standards

- Follow existing code style and naming conventions
- Add comments for complex logic
- Remove dead/commented-out code before submitting
- Use meaningful variable and function names

## Commit Messages

- Use clear, descriptive commit messages
- Start with a verb in imperative mood (e.g., "Add", "Fix", "Update")
- Reference issue numbers where applicable (e.g., "Fixes #123")

## Getting Started

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-feature`)
3. Make your changes
4. Run tests locally
5. Open a pull request

Thank you for helping improve this project!
